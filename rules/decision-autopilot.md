# Decision Autopilot Rule (의사결정 자동조종)

> 모든 의사결정 분기점(기획·기술·메타)에서 자동 적용. 의도 탐지 + 듀얼 추천 + 자율 판단으로 사용자 의사결정을 보조하고, 자율 레벨 데이터를 축적합니다.
>
> **paths 미지정** — 모든 세션의 모든 분기점에서 작동해야 하므로 매 세션 무조건 로드. 본문은 이 비용에 비례해 짧게 유지(~95행 목표). 상세(L2/L3 동작, 강등 모니터링)는 [decision-autopilot-detail.md](.claude/rules/decision-autopilot-detail.md)에서 필요 시점에만 로드.

## 1. 세션 시작 시 (자동)

분기점 가능성 작업 시작 시:

1. `dev/docs/digital-twin/decision-profile.md` 읽기 — 성향 + AI 판단 규칙
2. `dev/docs/digital-twin/autonomy-validation.md` 읽기 — 현재 자율 레벨 + 일치율
3. 현재 자율 레벨에 맞는 동작 모드 적용 (L0~L1은 §2, L2/L3은 detail 파일)

## 2. L0~L1 동작 (현재): 의도 탐지 + 듀얼 추천 + 사용자 승인

분기점 발생 시:

**Step 0: 의도 탐지** — 추천 생성 전에 1줄로 추론. 형식: `🎯 탐지 의도: ~~~`. 활성 스킬에 `## Pitfalls`가 있으면 참조.

**Step 1: 듀얼 추천** — 의도 기반으로 두 가지 병렬 표시:

- **🧠 AI 독립 추천**: 프로파일 미참조, 도메인 지식·일반 원칙만으로 판단. `[🧠 AI 독립 추천]` 표기.
- **👤 프로파일 추천**: `decision-profile.md`의 성향 태그 + AI 판단 규칙 적용. `[👤 프로파일 추천: 태그명]` 표기.
- **일치 시**: `🎯 AI+프로파일 일치 추천` 통합 표기 (고신뢰).
- **불일치 시**: 양쪽 근거를 각각 명시 → 사용자가 선택.

**Step 2: 의도 검증** — 사용자 선택 후, 선택+근거와 탐지 의도의 정합성을 내부 평가 (✅/🔄/❌). 세션 종료 시 `/profile-update`에서 기록.

## 3. 분기점 인식 기준

다음을 모두 분기점으로 인식 (기획·기술·메타 무관):

- **기획**: 수치/구조/정책 결정, 복수 옵션 중 선택
- **기술**: 라이브러리·아키텍처 패턴·데이터 모델·에러 처리 전략 선택
- **메타**: 하네스 자산 추가/제거/변경, 워크플로우 변경, 자가-진화 루프 모든 단계

**보편 트리거**: AskUserQuestion 호출 자체가 분기점 인식의 명시적 신호.

**임계값** (사소한 결정 제외): 결정의 결과가 (a) 코드 1줄 이상 변경, (b) 사용자 워크플로우 영향, (c) 되돌리기 어려움 중 **하나 이상**에 해당할 때만 듀얼 추천 적용.

## 3.5 의도 탐지 기준

- **의도 ≠ 성향**: 성향은 안정적 가치관, 의도는 분기점별 구체 목표 (1줄, 행동 지향).
- **의도 ≠ 근거**: 근거는 선택 후 설명, 의도는 선택 전 목표.
- **사후 검증**: 사용자 선택+근거와 의도의 정합성 평가 (✅ 정확 / 🔄 부분 정확 / ❌ 오탐지).

## 4. 세션 종료 시 리마인드

세션 종료 감지 시 **분기점이 1건 이상 누적된 경우**:

> "이번 세션에서 N개의 분기점이 있었습니다. `/profile-update`로 의사결정 로그를 저장하시겠습니까?"

## 5. 자율 레벨 (현재 L0~L1)

| 레벨  | 동작                                       | 본문 위치                                               |
| ----- | ------------------------------------------ | ------------------------------------------------------- |
| L0~L1 | 의도 탐지 + 듀얼 추천 + 사용자 승인 (현재) | 본 파일 §2                                              |
| L2    | 자율+확인: 자율 결정 후 보고               | [detail.md](.claude/rules/decision-autopilot-detail.md) |
| L3    | 자율+예외: 일치 분기점 자율, 불일치만 보고 | [detail.md](.claude/rules/decision-autopilot-detail.md) |

> **자가-진화 루프 예외**: `self-improvement.md` "자가-진화 루프"의 모든 단계는 L2/L3에서도 항상 L1 동작(사용자 승인 필수) 유지.

## 6. 프로파일 충돌 시 우선순위

`decision-profile.md` §3 충돌 우선순위 테이블 적용. 테이블에 없는 충돌은 사용자에게 질문.

## 7. 자가 점검 (구조적 사각지대 방지)

AskUserQuestion 호출 후 즉시 자가 점검:

- 모든 옵션 description에 `[🧠 AI 독립 추천]` / `[👤 프로파일 추천: 태그]` / `[🎯 일치 추천]` 표기가 있는가?
- 의도가 question 본문 또는 첫 옵션 상단에 1줄로 표시되었는가?
- 누락 발견 시 → 사용자에게 사과 + 보정 질문 재호출.
