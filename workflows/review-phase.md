# Review Phase Workflow (P5~P6)

이 워크플로우는 구현 완료 후 코드 리뷰, 문서 최신화, TDD 검증을 수행하는 마무리 과정입니다.

1. **코드 자체 리뷰 (P5)**
   - 구현이 완료된 코드를 자체적으로 검토합니다. (타입 안정성 체크, 예외 처리, `.claude/rules` 규칙 준수 여부)
   - 문제가 발견 시 즉각적으로 코드를 다시 수정합니다.

2. **문서 최신화 (P5)**
   - `dev/active/[기능명]/context.md` 에 SESSION PROGRESS 최종 상태 작성, `tasks.md` 의 구현 항목들 최종 체크 완료.
   - 신규 기능이나 인프라 변경이 있었을 경우 `.claude/rules` 또는 `dev/docs/` 시스템 아키텍처 문서에도 반영합니다.

3. **아카이브 처리 (P5)**
   - 현재 작업 기능이 완전히 마무리되었다면, `dev/active/[기능명]` 폴더 전체를 `dev/archive/[기능명]` 경로로 이동시킵니다.

4. **교차 검증 제안 (P6) (Human-in-the-Loop)**
   - 작업 규모가 크거나 (3개 파일 이상, 로직 대폭 수정, DB 스키마/API 변경 등), 추가 확인이 필요한 경우 사용자에게 교차 검증 진행 여부를 확인합니다.
