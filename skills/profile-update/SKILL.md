# Profile Update (의사결정 디지털 트윈 — 세션 커밋)

세션에서 발생한 의사결정 분기점을 memory 파일에 커밋하고, AI 자율 판단 정확도를 업데이트합니다.

## 워크플로우

### Step 1: 세션 분기점 추출

현재 세션의 대화를 분석하여 **분기점**을 추출합니다 (기획·기술·메타 무관).

**분기점 인식 기준**: `decision-autopilot.md` §3 참조. 요약하면:

- 기획: 수치/구조/정책/옵션 선택
- 기술: 라이브러리·아키텍처·데이터 모델·에러 처리 결정
- 메타: 하네스 자산 추가/제거/변경, 워크플로우 변경, 자가-진화 루프 단계
- 사용자가 AI 제안을 수정/거부한 순간 (모든 유형 공통)

**추출 형식** (분기점마다):

| 항목             | 내용                                                                          |
| ---------------- | ----------------------------------------------------------------------------- |
| 분기점           | 결정이 필요했던 주제                                                          |
| 탐지 의도        | 분기점에서 AI가 추천 전에 탐지한 사용자 의도 (1-line)                         |
| 제안된 옵션      | AI가 제시한 옵션 목록                                                         |
| AI 독립 추천     | 프로파일 미참조, 맥락·도메인 지식만으로 판단한 추천                           |
| 프로파일 추천    | decision-profile.md 성향 태그 기반 추천 + 적용된 태그                         |
| 일치 여부        | 일치 / 불일치                                                                 |
| 사용자 선택      | 사용자가 최종 선택한 옵션                                                     |
| 선택 근거        | 사용자가 밝힌 이유 (직접 언급 또는 맥락 추론)                                 |
| 의도 정확 여부   | 정확 / 부분 정확 / 오탐지 (선택+근거와 탐지 의도의 사후 정합성 판단)          |
| 승자             | AI / 프로파일 / 양쪽 / 둘 다 아님                                             |

### Step 2: decision-log.md 업데이트

`memory/decision-log.md`를 읽고 새 세션의 분기점을 추가합니다.

**형식**:

```markdown
## [날짜] 세션 제목

**세션 요약**: 1~2줄 요약

| #   | 분기점 | 탐지 의도 | 옵션 | 선택 | 근거 | 의도 정확? | 태그 |
| --- | ------ | --------- | ---- | ---- | ---- | ---------- | ---- |
| N+1 | ...    | ...       | ...  | ...  | ...  | ...        | ...  |

### 세션 메타 분석

- 가장 빈번한 태그: ...
- 방향 전환 횟수: ...
- AI 권장과 일치율: N/M = X%
- 의도 탐지 정확도: N/M = X%
- 의도 일관성: [일관 / 부분 일관 / 비일관]
```

### Step 3: autonomy-validation.md 업데이트

`memory/autonomy-validation.md`를 읽고 새 세션의 검증 로그를 추가합니다.

**업데이트 항목**:

1. 세션별 검증 로그 테이블 추가
2. **듀얼 일치율 재계산**:
   - AI 독립 일치율: AI 독립 추천과 사용자 선택의 일치율
   - 프로파일 일치율: 프로파일 추천과 사용자 선택의 일치율
   - **프로파일 부가가치**: 프로파일 일치율 - AI 일치율
3. 불일치 패턴 요약 업데이트
4. 레벨 전환 기준 추적 테이블 업데이트
5. **의도 탐지 정확도 재계산**
6. **의도 일관성 평가**

### Step 4: decision-profile.md 재보정 판단

**재보정 트리거 확인**:

| 트리거         | 조건                                 | 동작                                           |
| -------------- | ------------------------------------ | ---------------------------------------------- |
| 연속 거부      | AI 추천 3회 연속 불일치              | 해당 태그 AI 판단 규칙 재분석 → 프로파일 수정  |
| 새 패턴        | 기존 태그에 매핑 안 되는 분기점 2건+ | 새 성향 태그 생성 제안                         |
| 가중치 변동    | 특정 태그가 세션에서 3회+ 발현       | 해당 태그 가중치 재평가                        |
| 누적 20건 도달 | decision-log 전체 분기점 20건+       | 프로파일 전면 재분석 (가중치 재산정)           |
| 의도 패턴 발견 | 동일 유형 의도 3건+ 누적             | decision-profile.md 의도 패턴에 추가 제안       |

**재보정 불필요 시**: "프로파일 변경 없음" 보고 후 종료.
**재보정 필요 시**: 변경 사항을 사용자에게 제시 → 승인 시 `decision-profile.md` 수정.

### Step 5: 레벨 전환 제안

`autonomy-validation.md`의 누적 일치율이 전환 기준에 도달했는지 확인:

| 전환    | 조건                                                               | 동작                               |
| ------- | ------------------------------------------------------------------ | ---------------------------------- |
| L1 → L2 | 일치율 85%+ AND 의도 정확도 **90%+** / 20건+ / **5세션 연속 충족** | "자율+확인 모드 전환을 제안합니다" |
| L2 → L3 | 일치율 95%+ AND 의도 정확도 **95%+** / 40건+ / **5세션 연속 충족** | "자율+예외 모드 전환을 제안합니다" |

전환 제안 시 사용자 승인 필요.

### Step 5.5: 디지털 트윈 데이터 미러링

memory 핵심 파일 3개를 프로젝트 repo(`dev/docs/digital-twin/`)에 미러링합니다.

**안전 검증 (파일별 수행)**:

- memory 원본이 존재하고, 기존 미러보다 **파일 크기가 크거나 같으면** → 미러링 진행
- 원본 부재 또는 기존 미러보다 **크기가 작으면** → 해당 파일 미러링 **건너뛰기** + 사용자 경고

### Step 6: 완료 보고

```

세션: [세션 제목]

듀얼 추천 분석:
  AI 독립 일치율: 이번 [X1/N] / 누적 [Y1/M] = [Z1%]
  프로파일 일치율: 이번 [X2/N] / 누적 [Y2/M] = [Z2%]
  프로파일 부가가치: [Z2 - Z1]%p

의도 탐지:
  의도 정확도: 이번 [X/N] / 누적 [Y/M] = [Z%]
  의도 일관성: [일관 / 부분 일관 / 비일관]

프로파일 재보정: [변경 없음 / 태그 N개 업데이트]

파일 업데이트:
- memory/decision-log.md
- memory/autonomy-validation.md
- memory/decision-profile.md [수정 / 변경 없음]
- dev/docs/digital-twin/ 미러링
```
