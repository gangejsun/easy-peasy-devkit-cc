# 서비스 기획 프레임워크

SKILL.md에서 참조하는 서비스 기획 관련 프레임워크 및 템플릿입니다.

---

## 1. User Story 작성 가이드

### 형식

```
[역할]로서, [목표]하기 위해, [기능]하고 싶다.
```

### 좋은 User Story의 INVEST 기준

| 기준            | 의미                 | 확인 방법                              |
| --------------- | -------------------- | -------------------------------------- |
| **I**ndependent | 다른 스토리와 독립적 | 순서 없이 개발 가능한가?               |
| **N**egotiable  | 협의 가능            | 구현 방식이 고정되어 있지 않은가?      |
| **V**aluable    | 가치 있음            | 사용자나 비즈니스에 가치를 제공하는가? |
| **E**stimable   | 추정 가능            | 공수를 추정할 수 있는가?               |
| **S**mall       | 작은 단위            | 한 스프린트에 완료 가능한가?           |
| **T**estable    | 테스트 가능          | 수용 기준을 정의할 수 있는가?          |

## 2. MoSCoW 우선순위 가이드

| 우선순위   | 의미                    | 기준                 |
| ---------- | ----------------------- | -------------------- |
| **Must**   | 없으면 출시 불가        | MVP 핵심, 법적 요건  |
| **Should** | 중요하지만 출시 가능    | 주요 기능, UX 개선   |
| **Could**  | 있으면 좋지만 필수 아님 | 편의 기능, 부가 가치 |
| **Won't**  | 이번 범위에서 제외      | 향후 버전, 낮은 ROI  |

## 3. Acceptance Criteria 가이드

### Given-When-Then 형식

```
Given: [사전 조건]
When: [사용자 행동]
Then: [예상 결과]
```

### 체크리스트 형식

```
- [ ] [검증 항목 1]
- [ ] [검증 항목 2]
```

## 4. User Story 분해 가이드

```
User Story: "[역할]로서 [기능]을 원한다"
  → Epic: [대분류]
    → Feature: [중분류]
      → Requirement: FR-001, FR-002...
```

## 5. 비기능 요구사항 체크리스트

| 카테고리   | 주요 항목                                 |
| ---------- | ----------------------------------------- |
| 성능       | 응답 시간, 동시 사용자 수, 처리량         |
| 보안       | 인증, 인가, 데이터 암호화, 감사 로그      |
| 접근성     | WCAG 수준, 키보드 내비게이션, 스크린 리더 |
| 호환성     | 브라우저, 모바일, 해상도                  |
| 유지보수성 | 모듈화, 문서화, 테스트 커버리지           |
