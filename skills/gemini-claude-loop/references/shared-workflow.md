# AI Review Loop — 공통 워크플로우

이 파일은 codex-claude-loop과 gemini-claude-loop이 공유하는 워크플로우를 정의합니다.
provider별 차이(CLI 명령어, API 키, 전제조건)는 각 SKILL.md에서 정의합니다.

## 핵심 철학

- **Claude Code**: 아키텍처 설계, 구현, 수정
- **외부 AI**: 검증, 코드 리뷰, 품질 보증
- **지속적 교차 검증**: 각 AI가 상대의 작업을 리뷰

## 공통 Step

### Step 1: 계획 수립 (Claude)

1. 작업에 대한 상세 계획 작성
2. 구현 단계를 명확하게 분리
3. 가정사항과 잠재적 이슈 문서화

### Step 3: 피드백 루프

외부 AI가 이슈를 발견한 경우:

1. 외부 AI의 우려사항을 사용자에게 요약
2. 피드백 기반으로 계획 수정
3. 사용자에게 확인
4. 필요 시 Step 2 반복

### Step 4: 구현 (Claude)

계획 검증 완료 후:

1. Claude가 Edit/Write/Read 도구로 코드 구현
2. 구현을 관리 가능한 단위로 분리
3. 각 단계별 에러 처리 포함
4. 구현 내용 문서화

### Step 6: 반복 개선

1. 외부 AI 리뷰 후 Claude가 수정 적용
2. 중대한 변경 시 다시 외부 AI로 재검증
3. 코드 품질 기준 충족 시까지 반복

## 에러 처리

1. 외부 AI CLI 비정상 종료 코드 시 중단
2. 외부 AI 피드백을 요약하고 방향 결정
3. 중대한 아키텍처 변경, 다수 파일 영향, 하위 호환 깨지는 변경 시 사용자 확인 후 진행

## 모범 사례

- 구현 전 **반드시 계획 검증** 수행
- 변경 후 **교차 리뷰 생략 금지**
- AI 간 **명확한 핸드오프** 유지
- **누가 무엇을 했는지** 문서화
