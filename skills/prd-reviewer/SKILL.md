# PRD Reviewer

기존 PRD 문서의 품질을 진단하고, 반영 위치를 명시한 개선안을 도출하는 스킬입니다. 단일 PRD와 모듈형 PRD(prd-overload + Phase Sub PRD + Shared 문서) 모두 지원합니다.

## 워크플로우

### Step 1: 대상 PRD 결정 + PRD 유형 감지 + 진단 범위 선택

#### 1-1: 대상 PRD 결정

| 상황                   | 동작                                           |
| ---------------------- | ---------------------------------------------- |
| 사용자가 PRD 파일 지정 | 해당 파일로 진행                               |
| PRD 파일 미지정        | `dev/docs/prd/` 탐색 후 목록 제시, 사용자 선택 |

#### 1-2: PRD 유형 자동 감지

| 감지 조건                                               | PRD 유형   | 진단 방식         |
| ------------------------------------------------------- | ---------- | ----------------- |
| `prd-overload.md` 존재 + `phases/`, `shared/` 폴더 존재 | **모듈형** | 6축 진단 (A~F)    |
| 단일 `.md` 파일                                         | **단일**   | 5축 진단 (A~D, F) |

#### 1-3: 진단 범위 선택

| 진단 범위    | 설명                                                        | 실행 Step                           |
| ------------ | ----------------------------------------------------------- | ----------------------------------- |
| 종합 진단    | 전체 축 검증 (단일: A~D+F, 모듈형: A~F)                     | Step 2 A→B→C(→E)→F → Step 3→4       |
| 특정 섹션    | 사용자 지정 섹션만 집중 진단                                | Step 2 A→B→C (범위 한정) → Step 3→4 |
| 구현 대조    | 코드베이스 대비 괴리 중심                                   | Step 2 D → Step 3→4                 |
| 모듈형 구조  | prd-overload 기준 크로스 문서 정합성 (모듈형 전용)          | Step 2 E → Step 3→4                 |
| 수정 후 검증 | PRD 수정 직후 일관성 + 설계 원칙 검증. task-workflow 트리거 | Step 2 C→E(모듈형)→F → Step 3→4     |

### Step 2: PRD 품질 진단

대상 PRD를 읽고 진단합니다. 각 축에서 발견한 이슈에 **반드시 PRD 내 위치(섹션명 + 해당 내용)**를 기록합니다.

#### A. 구조적 완결성

#### B. 요구사항 명확성

#### C. 내부 일관성

#### D. 구현 정합성 (구현 대조 모드)

#### E. 모듈형 구조 정합성 (모듈형 PRD일 때만)

#### F. 설계 원칙 정합성 (단일·모듈형 공통)

## Pitfalls

- 축 F 검증 시 원칙의 문자적 해석에 머물지 말 것 — 구현에 미치는 구조적 영향까지 추적
- 용어 일관성 검증 시 grep 한 번으로 끝내지 말 것 — 약어·동의어·복합명사 변형까지 검색
- "적절한", "필요 시" 같은 모호한 표현 자체를 Critical로 올리지 말 것 — 구현에 실질적 영향이 있는지가 기준

### Step 3: 진단 보고서 생성

**심각도 기준:**

| 심각도    | 기준                              | 예시                                                     |
| --------- | --------------------------------- | -------------------------------------------------------- |
| Critical  | 구현 방향에 직접 영향, 모순/오류  | 수치 불일치, 상태 전이 모순, 용어 혼용, 끊어진 문서 링크 |
| Important | 명확성 부족으로 구현 시 혼란 가능 | 모호한 요구사항, 예외 케이스 누락, FR↔Screen 매핑 누락   |
| Minor     | 품질 개선, 가독성 향상            | 섹션 순서, 형식 통일, 사소한 오탈자                      |

### Step 4: PRD 업데이트 적용

사용자가 승인한 개선안만 PRD에 반영합니다.

### Step 5: 완료 보고

```
PRD 진단 및 업데이트가 완료되었습니다.

PRD 유형: [단일 / 모듈형]
진단 결과: Critical [N]건 / Important [N]건 / Minor [N]건
적용 항목: [N]건 / 전체 [M]건
문서 버전: [이전] → [현재]
역검증: [통과/잔존 N건]

파일: [PRD 파일 경로 또는 모듈형 구조 목록]
```
