# 사업 기획 프레임워크

SKILL.md에서 참조하는 사업 기획 관련 프레임워크 및 템플릿입니다.

---

## 1. 5C Analysis 가이드

5C 분석은 사업 환경을 체계적으로 이해하기 위한 프레임워크입니다.

| 요소 | 핵심 질문 | 분석 포인트 |
|------|----------|------------|
| **Company** (자사) | 우리의 핵심 역량은? | 기술력, 인적자원, 자산, 브랜드, 기존 사용자 기반 |
| **Customer** (고객) | 누구의 문제를 해결하는가? | 세그먼트, 니즈, 구매력, 의사결정 방식 |
| **Competitor** (경쟁) | 대안은 무엇이 있는가? | 직접 경쟁, 간접 경쟁, DIY 대안, 전환 비용 |
| **Collaborator** (협력) | 누구와 함께해야 하는가? | 공급자, 유통, 기술 파트너, 플랫폼 |
| **Climate** (환경) | 외부 환경은 어떤가? | PEST(정치/경제/사회/기술), 규제, 트렌드 |

**분석 팁:**
- Company는 프로젝트의 `CLAUDE.md`와 기존 코드베이스에서 자동 추출 가능
- Competitor는 `/research` (경쟁사 분석)의 결과를 통합
- Climate는 `/research` (시장 조사)의 트렌드 데이터를 통합

## 2. Lean Canvas 템플릿

9칸 구조로 사업 모델을 한 눈에 정리합니다.

```
┌─────────────────┬─────────────────┬─────────────────┐
│  ② 문제          │  ④ 솔루션        │  ③ 핵심 가치 제안  │
│  Top 3 문제      │  핵심 기능 3개    │  UVP 한 문장      │
│                 │                 │                 │
│  기존 대안:       │                 │  높은 수준 개념:    │
│  [현재 해결책]     │                 │  [한줄 설명]       │
├─────────────────┼─────────────────┼─────────────────┤
│  ⑧ 핵심 지표      │                 │  ⑤ 고객 세그먼트   │
│  측정할 핵심 활동   │                 │  타겟 고객       │
│                 │                 │  얼리 어답터:     │
├─────────────────┤                 ├─────────────────┤
│  ⑥ 채널          │                 │  ⑨ 경쟁 우위     │
│  고객 도달 경로    │                 │  모방 불가한 것    │
├─────────────────┴─────────────────┴─────────────────┤
│  ⑦ 비용 구조                  │  ① 수익원               │
│  고정비 + 변동비              │  수익 모델 + 가격        │
└──────────────────────────────┴──────────────────────┘
```

**작성 순서**: ① 수익원 → ② 문제 → ③ UVP → ④ 솔루션 → ⑤ 고객 → ⑥ 채널 → ⑦ 비용 → ⑧ 지표 → ⑨ 경쟁 우위

## 3. 가격 책정 프레임워크

### 3가지 접근 방식

| 방식 | 기준 | 적합한 상황 | 장점 | 단점 |
|------|------|------------|------|------|
| **Cost-based** | 원가 + 마진 | 비용 예측 용이, B2B | 이익 보장 | 시장 가치 반영 안됨 |
| **Value-based** | 고객이 느끼는 가치 | 차별화된 솔루션, SaaS | 높은 마진 가능 | 가치 측정 어려움 |
| **Competitor-based** | 경쟁사 가격 | 경쟁 치열, 레드오션 | 시장 수용성 높음 | 가격 경쟁 함정 |

### SaaS 가격 티어 설계 가이드

| 티어 | 역할 | 가격 설정 |
|------|------|----------|
| **Free** | 사용자 유입, 제품 체험 | 무료, 기능/용량 제한 |
| **Starter** | 전환 유도, 핵심 기능 | 경쟁사 하위 가격대 |
| **Pro** | 핵심 수익, 주력 타겟 | 가치 기반 + 경쟁 참조 |
| **Enterprise** | 대형 고객, 맞춤 | 개별 협상 |

### 가격 결정 체크리스트
- [ ] 고객이 현재 대안에 지불하는 비용은?
- [ ] 우리 솔루션이 절약/창출하는 가치는?
- [ ] 경쟁사 가격 범위는?
- [ ] 원가 대비 목표 마진은?
- [ ] 초기 할인/프로모션 전략은?

## 4. 리스크 매트릭스 템플릿

### 영향도 × 발생 확률 매트릭스

```
영향도 ↑
높음  │ 모니터링    │ 적극 대응   │ 최우선 대응  │
      │ (낮×높)    │ (중×높)    │ (높×높)    │
중간  │ 수용       │ 모니터링    │ 적극 대응   │
      │ (낮×중)    │ (중×중)    │ (높×중)    │
낮음  │ 수용       │ 수용       │ 모니터링    │
      │ (낮×낮)    │ (중×낮)    │ (높×낮)    │
      └───────────┴───────────┴───────────→ 발생 확률
        낮음         중간         높음
```

### 리스크 카테고리

| 카테고리 | 일반적 리스크 | 체크 포인트 |
|---------|-------------|------------|
| **시장** | 수요 부족, 경쟁 심화, 타이밍 | 시장 검증 방법, 경쟁 대응 |
| **기술** | 확장성, 보안, 기술 부채 | 아키텍처 검증, 성능 기준 |
| **운영** | 인력 부족, 번아웃, 커뮤니케이션 | 팀 구성, 프로세스 |
| **법규** | 개인정보보호, 라이선스, 금융 규제 | 규제 준수 체크리스트 |
