# Research

주어진 주제에 대해 공개 웹 정보를 수집하고 구조화된 리서치 문서를 생성하는 스킬입니다.

## 한계
- 유료 보고서, 비공개 API, 인증 필요 서비스의 데이터 수집
- 실시간 가격/사용자 수 등 정확한 수치 보장
- 한국어 로컬 데이터의 완전한 커버리지

**핵심 원칙:**
- AI가 "분석"하는 척하지 않는다 — 수집된 정보의 출처를 항상 명시
- 확인되지 않은 수치에는 `[미확인]` 태그를 붙인다
- 사용자가 직접 수집한 데이터가 있으면 WebSearch 결과보다 우선 사용

## 워크플로우

### Step 1: 리서치 유형 판별

사용자 요청에서 리서치 유형을 판별합니다:

| 유형 | 판별 키워드 | 출력 구조 |
|------|------------|----------|
| 경쟁사 분석 | 경쟁사, 경쟁 서비스, 대안, 유사 서비스 | 경쟁사별 프로필 + 비교표 |
| 시장 조사 | 시장, 규모, 트렌드, 성장률 | 시장 현황 + 트렌드 + 데이터 출처 |
| 기술 조사 | 기술, 비교, 라이브러리, 프레임워크 | 기술별 장단점 + 적합성 평가 |
| 사용자 조사 | 사용자, 니즈, 페인포인트, 타겟 | 페르소나 + 페인포인트 + 시나리오 |
| 자유 주제 | 기타 | 범용 조사 구조 |

**P0 파이프라인 경유 시:**

| 호출 상황 | 실행 Step |
|----------|----------|
| P0 파이프라인 호출 (planning-agent 경유) | P0-A 결과에서 리서치 주제 자동 추출 → Step 4 → Step 5 → Step 6(P0-C 연결) |
| 사용자 직접 호출 | 아래 판별 절차대로 진행 |

판별이 모호하면 사용자에게 확인합니다.

### Step 2: 기존 리서치 확인

`dev/docs/research/`에서 관련 문서를 탐색합니다.

| 상황 | 액션 |
|------|------|
| 관련 문서 존재 | 업데이트할지 새로 만들지 사용자 확인 |
| 문서 부재 | Step 3 진행 |

### Step 3: 사용자 데이터 확인

사용자에게 직접 수집한 자료가 있는지 확인합니다:

```
이 주제에 대해 직접 수집하신 자료(스크린샷, 메모, 링크, 파일 등)가 있나요?
있으면 공유해주시면 리서치에 통합합니다. 없으면 공개 웹 검색으로 진행합니다.
```

| 상황 | 동작 |
|------|------|
| 사용자 자료 있음 | 자료를 기반으로 구조화 + WebSearch로 보완 |
| 사용자 자료 없음 | WebSearch만으로 진행 (한계 명시) |

### Step 4: 정보 수집

**검색 전략:**
1. 주제에서 3-5개 검색어를 도출합니다
2. 한국어와 영어 검색어를 모두 활용합니다
3. 검색어 예시: `"AI dev workflow"`, `"Claude Code harness"`, `"AI native development platform"`

**수집 프로세스:**
```
WebSearch로 각 검색어 실행
→ 상위 결과에서 유의미한 페이지 선별 (블로그보다 공식 사이트/보도자료 우선)
→ WebFetch로 주요 페이지 상세 내용 수집
→ 출처 URL과 접근 날짜를 반드시 기록
```

**수집 기준:**
- 공식 사이트 > 보도자료 > 리뷰 기사 > 블로그 (신뢰도 순)
- 최신 정보 우선 (1년 이내)
- 수치 데이터는 출처가 명확한 것만 채택, 나머지는 `[미확인]` 태그

### Step 5: 구조화된 문서 생성

`resources/research-templates.md`의 리서치 유형별 템플릿을 참조하여 문서를 작성합니다.

**모든 유형 공통 규칙:**
- 모든 수치/사실에 출처 URL 필수 명시
- 확인되지 않은 정보에 `[미확인]` 태그
- "한계 및 추가 조사 필요 사항" 섹션으로 WebSearch의 한계를 명시적으로 기록
- "PRD 반영 제안" 섹션으로 이 리서치가 개발에 어떻게 활용될 수 있는지 제안

### Step 6: 파일 저장 및 사용자 확인

**저장:**
- 저장 경로: `dev/docs/research/<topic>.md`
- 파일명 규칙: kebab-case (예: `competitor-analysis.md`, `market-research.md`, `realtime-notification-tech.md`)

**사용자 확인:**
```
리서치 문서가 생성되었습니다.

📄 파일 위치: dev/docs/research/<topic>.md

## 리서치 요약
[핵심 발견 3-5개]

## 신뢰도
[전체 신뢰도 평가와 근거]

## 추가 조사 필요 사항
[WebSearch로 확인 불가한 핵심 항목]

---
이 리서치를 PRD에 반영하려면:
"PRD 만들어줘. dev/docs/research/<topic>.md 참조해서"
```

### P0 파이프라인 연결 시 후속 안내

planning-agent 경유 시:
- 리서치 결과를 planning-agent에게 반환
- planning-agent가 P0-C(사업 기획) 진행 여부를 사용자에게 확인

```
리서치가 완료되었습니다.

📄 파일 위치: dev/docs/research/<topic>.md

다음 단계로 사업 기획(P0-C)을 진행할 수 있습니다.
사업 기획으로 진행하려면: /business-planner
```

- 리서치 저장소: `dev/docs/research/`
