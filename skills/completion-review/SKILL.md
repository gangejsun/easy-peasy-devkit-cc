# Completion Review

개발 완료 후 문서 최신화 및 아카이브를 수행하는 스킬입니다.

## 워크플로우

### Step 1: 작업 유형 확인 및 업데이트 범위 결정

현재 작업 유형에 따라 문서 업데이트 범위를 결정합니다.

| 작업 유형 | Step 2 (개발 문서) | Step 3 (프로젝트 문서) | Step 4 (아카이브) |
|----------|------------------|---------------------|-----------------|
| 신규 기능 개발 | context + tasks 전체 업데이트 | PRD, CLAUDE.md, dev/docs/ 전체 확인 | 수행 |
| 기존 기능 확장/수정/삭제 | context + tasks 업데이트 | 변경된 부분의 관련 dev/docs/ | 수행 |
| 버그 수정 | tasks만 (dev docs가 있는 경우) | 최소 업데이트 (필요한 경우만) | 생략 가능 |
| 리팩토링 | context + tasks 업데이트 | 아키텍처 문서 중점 업데이트 | 수행 |

### Step 2: 개발 문서 업데이트

`dev/active/<feature-name>/` 의 문서를 업데이트합니다:

| 파일 | 업데이트 내용 |
|------|-------------|
| `context.md` | SESSION PROGRESS 섹션을 완료 상태로 업데이트 |
| `tasks.md` | 모든 완료된 작업을 `[x]`로 체크 |

### Step 3: 프로젝트 문서 업데이트

구현 중 발생한 변경사항을 분석하고, 해당하는 문서를 업데이트합니다:

| 변경 내용 | 업데이트 대상 |
|----------|-------------|
| 새 API 엔드포인트 | 관련 `.claude/rules/` 파일 |
| DB 스키마 변경 | 관련 `.claude/rules/` 파일 (필요시) |
| 아키텍처 변경 | `dev/docs/architecture/` |
| API 스펙 변경 | `dev/docs/api/` |
| PRD와 다른 구현 | 해당 PRD 파일 업데이트 |
| 새 서비스/엔드포인트 추가 | `CLAUDE.md` Code Conventions 섹션 |

변경사항 분석 체크리스트:
- 추가/수정/삭제된 파일 목록 확인
- 새로운 API 엔드포인트가 추가되었는지 확인
- DB 스키마 변경사항이 있는지 확인
- 기존 인터페이스가 변경되었는지 확인

### Step 4: 아카이브

기능이 완전히 완료되면 이 스킬의 호출자(review-agent)가 직접 아카이브합니다:

```bash
git mv dev/active/<feature-name> dev/archive/<feature-name>
```

아카이브 전 확인사항:
- [ ] tasks.md 의 모든 항목이 체크되었는가?
- [ ] context.md 가 최종 상태를 반영하는가?
- [ ] 관련 프로젝트 문서가 모두 업데이트되었는가?

### Step 4.5: 산출물 자체 검증

Step 2~4 실행 후, 사용자 보고 전 다음을 확인:

| 검증 항목 | 기준 | 실패 시 |
|----------|------|--------|
| context.md 업데이트됨 | SESSION PROGRESS가 완료 상태 반영 | 재업데이트 |
| tasks.md 전체 체크 | 미체크 항목 없음 | 누락 항목 확인 후 체크 또는 미완료 사유 기록 |
| 관련 문서 업데이트 | Step 3 대상이 모두 처리됨 | 누락 문서 업데이트 |

### Step 5: 사용자에게 완료 보고

```
개발 완료 및 문서 최신화가 완료되었습니다.

📝 업데이트된 문서:
- dev/active/<name>/context.md → 완료 상태
- dev/active/<name>/tasks.md → 전체 체크 완료
- [추가 업데이트된 문서 목록]

📦 아카이브: dev/archive/<name>/

추가 수정이 필요하면 말씀해주세요.
```

## 프로젝트 커스텀 리소스

> 아래 경로에 `.md` 파일이 존재하면 완료 검토 시 자동으로 참조됩니다. 파일이 없으면 이 섹션은 무시됩니다.

| 카테고리 | 경로 | 이 스킬에서의 용도 |
|---------|------|-------------------|
| 코딩 표준 | `.claude/resources/coding-standards/` | 문서 품질 기준, 코드-문서 일관성 검증에 활용 |
